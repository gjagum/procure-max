# Domain Model

## ID Conventions

All entity IDs are strings with a prefix pattern for readability and routing.

| Entity | Format | Example | Generated by |
|---|---|---|---|
| Users | `user:<guid N>` | `user:a1b2c3d4e5f6...` | Server (seeded admin: `user:admin`) |
| Roles | `role:<key>` (system) or `role:<guid N>` (custom) | `role:admin`, `role:requestor` | Server |
| Permissions | `perm:<area>:<action>` | `perm:dashboard:view` | Seeded |
| Cost Centers | `cc:<UPPERCODE>` | `cc:CC-1000` | Server from `code` |
| GL Accounts | `gl:<CODE>` | `gl:5000-OFFICE` | Server from `code` |
| Units | `unit:<CODE>` | `unit:EA` | Server from `code` |
| Suppliers | `supplier:<guid N>` | `supplier:abc...` | Server |
| Items | `item:<guid N>` | `item:def...` | Server |

Code-based entities derive their ID from the uppercased business code. GUID-based entities generate a new GUID on insert. Client-supplied IDs in request bodies are ignored.

## Money Value Object

The `Money` readonly record struct is the universal monetary type. It stores amounts as `long` minor units (cents).

```csharp
// Construction
var price = Money.FromDecimal(19.99m, "USD");  // → MinorUnits = 1999
var zero  = Money.Zero("USD");                   // → MinorUnits = 0

// Arithmetic
var total = price * 3;             // → 59.97 USD
var sum   = price + anotherPrice;  // → preserves currency
var diff  = price - discount;      // → preserves currency

// Comparison
if (price > minPrice) { }

// Display
price.ToString();  // → "19.99 USD"
```

**Rules:**
- Mismatched currency codes throw `InvalidOperationException`
- Negative multipliers throw `InvalidOperationException`
- Currency is always explicit — there is no "unitless" Money
- Arithmetic does not drift — all operations work on `long` minor units

## Purchase Requisition (Domain Aggregate)

The `PurchaseRequisition` class enforces procurement workflow rules in-memory, without a database.

### States

```
                    ┌──────────┐
                    │  DRAFT   │
                    └────┬─────┘
                         │ Submit (needs ≥1 line)
                         ▼
              ┌──────────────────┐
        ┌─────│ PENDING APPROVAL │─────┐
        │     └──────────────────┘     │
        │            │                 │
     Cancel       Approve           Reject
        │            ▼                 │
        │     ┌──────────┐            ▼
        │     │ APPROVED │      ┌──────────┐
        │     └──────────┘      │ REJECTED │
        │                       └─────┬────┘
        │                             │ Reopen
        │                             ▼
        │                       ┌──────────┐
        └──────────────────────→│ CANCELLED│
                                └──────────┘
```

### Business Rules

| Rule | Where Enforced |
|---|---|
| RequestorId is required | Constructor |
| Must have ≥1 line to submit | `Submit()` |
| Lines are immutable after submission | `EnsureMutable()` |
| Cannot self-approve | `Approve()` — `approverId != RequestorId` |
| Cannot self-reject | `Reject()` — `approverId != RequestorId` |
| Line currency must match PR currency | `AddLine()` — `EnsureSameCurrency()` |
| Quantity must be > 0 | `AddLine()` |
| Only rejected PRs can be reopened | `Reopen()` |
| Already-closed PRs cannot be cancelled | `Cancel()` |

### RequisitionLine

A line item within a requisition:

```csharp
record RequisitionLine(string Id, string ItemId, int Quantity, Money UnitPrice, string Uom)
```

- `LineTotal` = `UnitPrice * Quantity` (computed property)
- Lines are replaced, never mutated (immutable value type)

## Master Data Entities

All master data entities (Cost Centers, GL Accounts, Units, Suppliers, Items) follow the same pattern:

| Property | Description |
|---|---|
| `Id` | String ID per conventions |
| `Code` | Business code (unique) |
| `Name` | Display name |
| `IsActive` | Active flag |
| `CreatedAt` / `UpdatedAt` | Timestamps |
| `RowVersion` | Optimistic concurrency token (GUID `N` format) |
| `IsDeleted` | Soft-delete flag |

Suppliers additionally have: `LegalName`, `TaxId`, `ContactName`, `Email`, `Phone`, `Address*`, `PaymentTerms`, `Currency`, `IsBlocked`.

Items additionally have: `Sku`, `Description`, `Category`, `UnitId`, `GlAccountId`, `DefaultSupplierId`, `DefaultPriceMinor`, `DefaultCurrency`.
